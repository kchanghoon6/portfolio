SENTRY — Low-Cost Bridge Anomaly Detection

Developer (with an ML mentor) 한국코드페어 (Korea Code Fair) 2026 Python · PyTorch · Raspberry Pi

SENTRY is a low-cost, always-on screening system that flags abnormal vibration in small and medium bridges. Original title: "온습도 변화에 강인한 저비용 교량 진동 이상 탐지 시스템" — a bridge vibration anomaly-detection system robust to temperature and humidity change. It is the most substantial project I'm currently building, developed for the 2026 Korea Code Fair as the first stage of a roadmap toward ISEF.

The problem

Small and medium bridges — pedestrian overpasses and minor road bridges — are a monitoring blind spot. Class-3 bridges alone are about 64.5% of all bridges in Korea, yet they mostly rely on periodic visual inspection. Damage like corrosion, cracking and fatigue accumulates continuously, but inspections only sample the structure every few years, so changes between inspections can go unseen. The 2023 Jeongja Bridge collapse (Bundang, 1 death) and the 2022 Fern Hollow Bridge collapse (Pittsburgh) both involved damage that built up undetected between inspections — showing that "passed its last inspection" is not enough.

What SENTRY does

SENTRY is a first-line screening (auxiliary) system, not a replacement for precise inspection. Using a low-cost MEMS accelerometer (ADXL355Z), a temperature/humidity sensor (DHT22) and a Raspberry Pi 3 Model A+, it continuously watches a bridge's vibration, removes the part of the change that weather explains, and flags the remaining anomaly candidates — so limited inspection budget can be aimed at the bridges that most need a precise look.

How it works

  • Vibration sensing & features: a 3-axis MEMS accelerometer samples at 200 Hz; FFT/STFT extract per-window spectral features (RMS, peak frequency and power, band energy) and track the bridge's first natural frequency.
  • Environmental compensation: a healthy bridge's natural frequency naturally drifts with temperature and humidity. A multivariate regression learns that normal relationship, and anomaly detection runs on the residual — sharply cutting false alarms caused by weather rather than damage.
  • Unsupervised autoencoder: trained only on normal-state residual features; a reconstruction error above a 99th-percentile threshold marks an anomaly candidate. Because it never needs damage data, it can start screening from day one.
  • Explainable, band-wise alarms: instead of one opaque score, SENTRY reports which frequency band or mode drove the anomaly (band-wise reconstruction-error contribution, with optional SHAP) — so an inspector knows where to look first.

Validation

SENTRY is validated on two tracks: the public Z24 Bridge Benchmark (KU Leuven), a widely used vibration-based SHM dataset with progressive-damage tests, and a custom foam-board bridge model where mass-distribution changes simulate structural change. The environmental-compensation step is cross-checked against the openLAB Research Bridge long-term monitoring dataset, so the same pipeline is tested on both public data and hardware I built and recorded myself.

My role

I designed and built the system end to end — the sensing hardware, the Python data logger, the signal-processing and environmental-compensation pipeline, the autoencoder detector, and the experiments — working alongside a machine-learning mentor.
